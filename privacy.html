Privacy Policy

1. Introduction

Welcome to Code X Security! This Privacy Policy explains how Code X Security our bot collects, uses, maintains, and discloses information collected from users of the Code X Security Discord bot. We are committed to protecting your privacy and ensuring a safe and secure experience.

2. Information We Collect

Code X Security collects minimal data necessary to provide its features and improve your experience. The types of information we may collect include:

  • Discord User IDs: To identify individual users for features like AFK status, auto-responders, or specific command permissions.
  • Discord Guild (Server) IDs: To link configurations and settings to specific Discord servers where the bot operates (e.g., welcome messages, auto-responder rules).
  • Discord Channel IDs: For features that require specific channels, such as welcome message channels or logging channels.
  • Custom Messages/Content: If you configure features like custom welcome messages, auto-responder replies, or AFK messages, this content is stored to be used by the bot as intended.
  • Feature-Specific Data: Data related to specific functionalities you enable, such as AFK status (e.g., your AFK message, timestamp), or auto-responder rules (e.g., trigger phrases, responses).

3. How We Use Your Information

The information we collect is used solely for the purpose of providing and improving the functionality of Code X Security:

  • To enable and manage features you configure (e.g., welcome messages, auto-responders, AFK system).
  • To personalize your experience with the bot.
  • To maintain the bot's functionality and stability.
  • To provide support and respond to your inquiries.

4. Data Storage and Retention

We store your data only for as long as it is necessary to provide you with the services you use or until you explicitly request its removal.

  • General Data Retention: Data related to features you set up (like auto-responders, AFK messages, or server configurations) is stored persistently to ensure the bot functions correctly across sessions and restarts. This data is retained until you, the user, explicitly disable or remove the corresponding feature using the bot's commands.
  • Welcome Setup: If you configure a welcome setup for your server, the associated data (e.g., welcome message, channel ID) is stored. If you reset or disable the welcome setup via the bot's commands, this specific data will be removed from our database.
  • Auto-Responder System: When you set up auto-responder rules (e.g., trigger phrases and responses), this data is stored to allow the bot to function as intended. This data remains stored until you remove the specific auto-responder rule(s) using the bot's commands.
  • AFK System: If you set an AFK status, your AFK message and status are stored. This data is removed automatically when you return from AFK or explicitly clear your AFK status using the bot's commands.
  • User-Initiated Deletion: You can typically remove your data associated with specific features by using the bot's commands to disable or reset those features. For comprehensive data removal requests, please contact us using the information below.
